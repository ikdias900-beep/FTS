# RDR-0001: Repository License

## Status

APPROVED

## Intent

Select a final software, data, and documentation licensing strategy before public release.

## Options

1. Permissive open-source license for code plus explicit data/documentation licenses.
2. Copyleft license for code plus explicit data/documentation licenses.
3. Private repository until publication and archival terms are decided.

## AI Explanation in Plain Language

A license defines what other people may legally do with the code and artifacts. Choosing one is a publication and governance decision, not an implementation detail.

## Risks

- A permissive license may allow broad reuse with limited reciprocity.
- A copyleft license may limit adoption in some settings.
- No license makes public reuse legally unclear.

## Human Decision

Use the MIT License for project-authored repository contents, including code, documentation, and generated release data unless a file explicitly states different third-party terms.

Copyright notice:

```text
Copyright (c) 2026 Fitness, Truth & Structure Lab contributors
```

## What This Does Not Mean

This does not relicense external primary-source papers, publisher content, third-party packages, or any other material not authored for this repository.

## When To Revisit

Before adding third-party assets, archived source PDFs, externally contributed datasets, or publication assets requiring a more specific license.
